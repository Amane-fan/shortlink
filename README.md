# shortlink

一个面向高并发访问场景的短链接系统，覆盖短链接创建、跳转、分组管理、回收站、访问统计等核心能力，并在实现中引入分表、缓存、布隆过滤器、延迟任务、流式消费、网关路由和服务注册发现等工程化设计。

## 项目背景

短链接看起来是一个简单业务，但它天然同时具备以下特点：

- 读多写少，短链接跳转存在明显的热点访问场景
- 链接创建、修改、删除、统计等操作需要完整的数据模型支撑
- 访问链路短，但对吞吐、延迟和可用性要求高
- 很适合承载缓存、分库分表、异步处理、访问分析等后端常见技术点

因此，这个项目并不是单纯实现一个“长链接转短链接”的工具，而是把短链接系统中更接近真实业务的一些核心问题落到了代码里。

## 项目价值

- 业务模型清晰，面试时容易快速讲清楚背景、流程和核心价值
- 技术点集中，能够自然展开缓存、分表、异步削峰、统计链路、网关与微服务治理
- 系统复杂度适中，不会停留在简单 CRUD，也不会因为过度设计导致无法说明白

- 如何设计一个高频访问的短链接跳转系统
- 如何处理短链接创建和跳转场景下的数据一致性与查询性能
- 如何在项目里实际使用 Redis、ShardingSphere、消息流处理与服务拆分
- 如何在一个中等复杂度系统中划分模块边界

## 核心能力

- 短链接创建、修改、分页查询
- 短链接跳转与基础风控控制
- 分组管理与回收站能力
- 访问统计：PV、UV、UIP 及部分终端维度分析
- 基于 Redis 的缓存、布隆过滤器、延迟队列、Stream 消费
- 基于 ShardingSphere 的分表设计
- 支持网关路由、服务发现和聚合部署模式

## 技术亮点

### 1. 高并发读场景下的短链接跳转

短链接系统的核心压力集中在跳转请求。项目中围绕跳转链路引入了缓存、布隆过滤器、读写锁、延迟统计等机制，目标是减少数据库直连压力，并尽量缩短用户跳转路径。

### 2. 分表设计

项目使用 ShardingSphere-JDBC 对用户、分组、短链接等核心表进行分表处理，降低单表数据量增长带来的查询和维护压力。数据库脚本也已经按分片结构完整提供，便于本地复现。

### 3. Redis 多场景使用

Redis 在这个项目里不是单一做缓存，而是承担了多种职责：

- 热点数据缓存
- 布隆过滤器防缓存穿透
- 分布式锁
- 延迟队列
- Stream 消息处理

这种设计比较适合在面试中展开讲“一个中间件在不同业务场景下的落地方式”。

### 4. 聚合模式与拆分模式并存

项目保留了 `gateway`、`admin`、`project` 这类服务边界，同时又提供 `aggregation` 聚合模式，兼顾本地调试效率和服务拆分后的结构清晰度。

### 5. 可继续扩展

当前实现已经具备一个完整项目骨架，后续可以继续扩展：

- 自定义短码
- 更细的风控与限流
- 链接失效和续期策略
- 后台运营分析能力
- Docker Compose / 容器化部署
- 压测与自动化测试

## 架构说明

项目采用多模块结构：

- `gateway`：统一流量入口，负责路由和部分鉴权逻辑
- `admin`：后台管理接口，负责用户、分组、回收站、控制台相关能力
- `project`：短链接核心业务服务，负责创建、跳转、统计等核心链路
- `aggregation`：聚合服务，将 `admin` 和 `project` 合并运行，便于本地快速启动
- `console-vue`：前端控制台

默认情况下，网关走聚合模式，因此本地联调成本相对较低。

## 数据初始化

数据库脚本位于：

- `resources/database/link.sql`
- `resources/database/link-data.sql`

初始化示例：

```sql
CREATE DATABASE link DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

```bash
mysql -h127.0.0.1 -P3306 -uroot -p135790 link < resources/database/link.sql
mysql -h127.0.0.1 -P3306 -uroot -p135790 link < resources/database/link-data.sql
```

## 启动方式

本地推荐使用聚合模式，流程尽量保持简单：

1. 启动 MySQL、Redis、Nacos
2. 启动 `aggregation`
3. 启动 `gateway`
4. 如需页面，再启动 `console-vue`

默认端口：

- `gateway`: `8000`
- `project`: `8001`
- `admin`: `8002`
- `aggregation`: `8003`
